# Security policy

Please report suspected vulnerabilities privately through the repository's GitHub Security Advisories page. Do not open a public issue containing exploit details, customer data, credentials, or access tokens.

Include the affected endpoint or component, reproduction steps, impact, and any suggested mitigation. Maintainers should acknowledge a report within three business days, validate severity, prepare a fix on a private branch, and coordinate disclosure after supported deployments are patched.

Only the latest deployment from `main` is supported with security updates.
